调度与质量

前两个时间直接选择即可，系统会在保存时自动转换成 Cron。后面的参数用于控制选题数量和质量门槛。

每天自动执行一次健康检查。
每天自动执行一次主流程任务。

进入二次重排的候选主题数量。
文章质检达到该分数后才视为通过；否则会继续优化或走兜底策略。
选题时效策略 三档窗口自动取舍
新闻快讯：3 天内优先，7 天后若缺乏持续价值会被过滤。
产品发布与评测：7 天内优先，约 21 天后若不够深或不够有用会被过滤。
技术拆解与教程：30 天内基本正常参与，60 天后若不具备常青价值才会被过滤。
系统会综合新鲜度、价值分、深度分和常青价值分来判断，而不是只看发布时间。

模型配置（每个模型独立 API）

按模型职责拆分配置。若供应商选择阿里云百炼，只需填写 API 密钥并选择模型名称；若选择其他供应商，则继续按接口地址、API 密钥、模型 ID 自定义。

决策模型

?
负责工作流中的判断、质检与事实压缩。它会参与主题重排、文章质量评估，以及 FACT_COMPRESS 步骤中的正文事实提纯，所以更适合选择稳定、结构化输出能力较强的模型。
服务步骤：`RERANK`、`QUALITY_CHECK`、`FACT_COMPRESS`
已内置百炼专用地址，保存时将自动写入 DashScope 兼容 URL。当前同一套配置会同时服务 `RERANK / QUALITY_CHECK / FACT_COMPRESS` 等分析型步骤，系统会按输入复杂度自动放大超时档位。

重排模型

?
负责对候选话题或候选结果进行二次排序，让更适合公众号文章的内容优先排在前面。
若选择百炼专用重排模型，系统会自动切换到原生排序接口，无需你再手填 URL。正文感知重排时，候选数量和摘样长度越大，超时档位会自动提高。

写作模型

?
负责生成文章正文内容，把选题、结构和写作要求扩展为完整可发布的文案。
正文模型建议优先选择 `qwen-plus` 或 `qwen-max`，可在效果与成本之间灵活切换。系统不会裁剪正文质量所需输入，而是根据 prompt 复杂度自动提高超时档位。

封面提示词模型

?
负责把文章内容转换为高质量封面提示词，给图像模型提供更明确的视觉生成描述。
封面提示词通常用轻量文本模型就够了，百炼模式下可直接从常用模型中选择。

封面图片模型

?
负责根据封面提示词生成最终封面图或配图，直接决定文章对外展示时的视觉效果。
若选择百炼图片模型，保存后系统会自动走原生文生图接口；若改用其他平台，切回自定义即可。
当前已支持自动识别百炼原生能力：`qwen3-rerank` / `qwen3-rerank-plus` 会走排序接口，`wan2.6-t2i`、`wanx2.0-t2i-turbo` 等出图模型会走文生图接口；其他供应商保持原有自定义接入方式。
